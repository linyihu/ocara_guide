Upload

  

一、插件介绍

       文件上传插件，可以很方便地实现图片等文件的上传。

特色：

使用方便，还可以配置上传规则。

二、应用实例

$uploadObj = new Upload();

//设置保存目录，不存在时会自动新建该目录
//这里目录为public/temp/upload/item下面的当前商品ID目录（$item_id）
$path = 'upload/item/' . $item_id;
$uploadPath = ocPath('temp', $path);

if ($uploadObj->setSavePath($uploadPath) == false) {
	Error::show('图片保存目录新建失败!');
}

//设置上传文件类型
$uploadObj->setRules(array(
	'jpg', 'jpeg', 'gif', 'png'
));

//上传文件
if ($result = $uploadObj->upload()) {
   $thumb = ocGet('thumb.save_path', $result);
   $image = ocGet('image.save_path', $result);
} else {
   //输出上传错误
   Error::show($uploadObj->getError(), '返回');
}

//使用上传文件
$condition = array('id'	=> $id);
$data = array(
   'thumb' => $thumb,
   'image' => $image
);

//保存到数据库
$this->model->update($condition, $data);

实例结果：
	（1）$thumb和$image变量存放的都是保存的图片文件路径。
	（2）图片的文件名是根据框架当前时间戳自动生成唯一的随机字符，所以文件名没有重复性问题。
	（3）本例中使用了setSavePath()设置保存目录，setRules()设置文件类型。
	（4）表单必须是POST方式提交，并且使用了enctype="multipart/form-data"属性。这个可以自己添加，也可以使用表单的upload()方法。

三、方法函数介绍

1、setSavePath(savePath[, prefix])
  设置上传文件要保存的目录。  
  参数说明：
名称 描述
savePath 必需。保存目录。
prefix 可选。保存文件名的前缀。 
2、setRules(rules)
  设置允许上传的文件类型。  
  参数说明：
名称 描述
rules 必需。允许上传的文件类型数组。 
3、existsFile()
  检查是否存在上传的文件。 
4、upload(required)
  上传文件。  
  参数说明：
名称 描述
required 可选。是否打印错误。默认为false 
5、clear(path)
  清理上传过的文件。  
  参数说明：
名称 描述
path 必需。目录路径。 
6、errorExists()
  检测上传文件是否出错。 
7、setError(type[, params])
  设置错误信息。
