数据模型（Model）


1、新建模型
      新建模型有两种方法：
      （1）直接新建Model类。
      （2）使用Model类的build()方法。
      （3）使用控制器动作的model()方法，用于建立默认的Model。

2、详细说明
      （1）直接新建Model类
             实例如下：

//新建User模型类实例
$userModel = new \app\dal\models\User();

//新建BBS数据库连接下面的User模型类实例
$userModel = new \app\dal\models\BBS\User();



      （2）使用build()方法
          本框架所有的类，都继随自\Ocara\Base基类，该基类定义了build()方法，可以新增当前对象。
             实例如下：

//会新建application/model下面的User模型类实例
$userModel = \app\dal\models\User::build();

//会新建application/model/BBS下面的User模型类实例
$userModel = \app\dal\models\BBS\User::build();


      （3）使用控制器动作的model()方法
                  model([class])
             参数说明：


	

		名称
		描述
	

	

		class
		必需。数据模型类，必须是继承于框架的基本模型\Ocara\Core\Model。
		
	



             上述实例可以改成如下：

$userModel = $this->model('\app\dal\models\User');


             如果是bbs服务器默认库中的user表，则是：

$userModel = $this->model('\app\dal\models\BBS\User'); //会调用bbs数据库连接的目录中的User模型


             其他地方再次使用时，不会再重新建立该模型对象。如下示例：

class UpdateAction extends Controller
{
	public function _action()
	{}

	/**
	 * 初始化
	 */
	public function _display()
	{
        //用户表Model使用
		$userData = $this
                ->model('\app\dal\models\UserInfo') //第一次使用时，会新建该模型类对象
                ->limit(0, 100)
                ->getAll();

        //用户表Model使用
        $userData2 = $this
                ->model('\app\dal\models\UserInfo') //第二次使用，不会再新建类
                ->limit(0, 100)
                ->getAll();

        //home表Model使用
        $homeData = $this
            ->model() //因为home表模型名与控制名称同名，可以不传递参数建立模型
            ->limit(0, 100)
            ->getAll();
	}

    /**
     * 表单提交的处理
     */
	public function _submit()
	{}
}


 3、如何切换数据库和数据表（分库分表）？
      如果数据库要进行分库，方法如下：
      首先在Model类内部使用用_sharding()定义分库规则，然后外部调用sharing()方法传递分库规则要使用的具体数据，就可以进行分库。
      而在_sharding()中，可以使用selectDatabase()和selectTabe()切换数据库和表。
      请参考分库分表。
