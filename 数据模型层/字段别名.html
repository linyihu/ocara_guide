什么是字段别名

      
      字段别名，就是为了安全起见，设置的一个或多个字段的虚拟名称，用于表单元素起名和后台保存时自动获取值，以及作为查询条件时自动匹配字段。
      

如何指定字段别名


      只要新建一个model文件配置，其中定义$CONF['MAP']的键值数组的配置就行了，键名为字段别名，键值为字段名。
      比如，我们在required/model中添加user.php，在其中添加配置：

$CONF['MAP'] = array(
      'userName' => 'username',
      'password_alias' => 'password'
);

      当然你还可以定义某个字段的多个别名，如下username配置有两个别名：

$CONF['MAP'] = array(
    'userName' => 'username',
    'userName2' => 'username',
    'pass' => 'password'
);



字段别名有什么效果？


      在本框架中，字段别名，在添加或更新记录时，可以当作真实字段名用。
      而在查询时，真实字段会以字段别名输出。
      也就是说，字段别名可进库，出库也是字段名。但是在组成SQL语句中不需要用别名。
      （1）保存记录时字段别名当真实字段用
            比如说，下面我们定义了users表的几个别名。

$CONF['MAP'] = array(
      'userName' => 'username',
      'password_alias' => 'password'
);

            那么，我们在添加或更新数据时，可以将别名当真实字段用：

$data = array(
      'userName' => 'Jim',
      'password_alias' => '123456'
);

$user = new \Model\Users();
$user->data($data)->save();


            这跟以下代码是同样的效果：

$data = array(
      'username' => 'Jim',
      'password' => '123456'
);

$user = new \Model\Users();
$user->data($data)->save();


      （2）查询记录时真实字段以字段别名输出，条件数组中别名也可以作为条件
            同上例别名配置。那么查询时，真实字段会以字段别名返回：

$user = new \Model\Users();
$result = $user->where(array('userName' => 'Jim'))
     ->fields('username')
     ->fields(array('password'))
     ->getRow();
ocPrint($result);


            以上代码，用了字段别名userName当条件，而且真实字段输出的是字段别名，打印结果如下：

Array ( [userName] => Jim [password_alias] => e10adc3949ba59abbe56e057f20f883e )




应用场景和实例


（1）作为数据保存时匹配字段
      有很多时候，我们为了数据库安全，不希望他人知道数据表的字段名，所以不想让表单元素的命名与字段名同名。
      但是这样的话，一般地我们在后台处理时，不得不重新赋值一下，如下面实例，表单有两个字段
      userName和pass，分别表示用户名和密码，然后表单提交后可以这样添加记录：

$userModel= new \Model\User();
$data = array(
    'username' => Request::getPost('userName'),
    'password' => Request::getPost('pass')
);

//添加记录
$userModel->data($data)->save();

      然而，如果我们指定userName、pass分别是username、password的别名，表单提交后，后台处理数据就方便多了。
      首先，在conf/model/user.php中指定别名：

$CONF['MAP'] = array(
    'userName2' => 'username',
	'pass' => 'password'
);

      然后，添加记录，就不需要如此麻烦了：

$userModel= new \Model\User();

//添加记录
$userModel->data($data)->save();

      由于系统会自动匹配字段名称与别名，所以直接使用data()，不需要赋值给字段就能保存。

（2）作为查询条件时匹配字段
       字段别名在关联数组中做查询条件时，也能匹配到。如将上例中POST提交的数据作为查询条件；

$userModel= new \Model\User();

//查询结果
$result = $userModel->where($_POST)->getRow();
