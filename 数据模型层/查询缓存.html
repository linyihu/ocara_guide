查询缓存


1、什么是查询缓存
      一般的数据库查询都是直接使用SQL从数据库去查询，对于一些静态或不常改变的数据，我们可以放到缓存，从而减少数据库查询，降低数据库负载。
      
2、如何使用查询缓存
      框架中要使用查询缓存，只要使用Model的cache()方法，它会将SQL语句时行md5()加密，保存到缓存。格式如下：
               cache([server, required])
      参数说明：


	

		名称
		描述
	

	

		server
		可选。缓存服务器的名称。默认是main。
	

	

		required
		可选。是否必须，默认为false。如果为true，那么如果扩展无法加载或不在时，会抛出错误提示。
	




      实例如下：

$where = array(
	'uid' => 21
);

//保存到默认缓存，假设默认是Memcache缓存连接
$userModel = new \app\dal\models\User();
$result = $userModel->where($where)
	->cache()
	->getAll();

//保存到Redis缓存，假设已经配置了Redis的缓存连接
$userModel = new \app\dal\models\User();
$result = $userModel->where($where)
	->cache('cache_redis')
	->getAll();

3、自定义缓存的保存和查询方式
         Model的cache()保存数据时是以md5加密SQL为键名的，键值是以json_encode()函数加密。
         如果要修改加密算法，可以使用事件回调。要在配置文件event.php中修改model选项。
         比如，保存用QueryCache类的save()方法，而查询时使用find方法。
         首先，保存时的回调方法代码为：

class QueryCache extends Base
{
	public function save($cacheObj, $sql, $ret, $required)
	{
		$value = serialize($ret);
		$key   = md5($sql . '123');
		
		$cacheObj->set($key, $value);
	}
}


         查询时的回调动作代码为：

class FindAction extends QuerycacheController
{
	/**
	 * 初始化
	 */
	public function __action($cacheObj, $sql, $required)
	{
		$key   = md5($sql . '123');
		$value = $cacheObj->get($key);
		$ret   = unserialize($value);
		
		//查询要返回查询数组，所以反序列化
		return $ret;	
	}
}


         然后，在event.php进行回调配置：

/*
 * 数据模型Model相关事件
 */
'model' => array(
	//Model查询缓存的事件
	'query' => array(
		'save_cache_data' => array('QueryCache', 'save'), //保存为缓存数据的事件
		'get_cache_data' => array('QueryCache', 'find'),  //获取缓存数据的事件
	)
),
