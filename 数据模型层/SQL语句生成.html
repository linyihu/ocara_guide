SQL查询语句的生成方法

 
 1、联接查询
      （1）左联接
            leftJoin(class, alias, on)
      （2）右联接
             rightJoin(class, alias, on)
      （3） 内全联接
            innerJoin(class, alias, on)
      联接查询可以使用关联配置（请参考数据模型层/关联模型操作），也可以在查询时自定义关联。
      参数说明：
名称 描述
class 必需。数据表模型类名或关联配置名。
（1）数据表模型类名，如\app\model\database\User;
（2）关联配置名，配置在application/config/model中的模型配置$CONF['JOIN']。
alias 可选。表的别名。
on 可选。联接条件，如果class是数据表模型类，本参数必需！ 
      实例如下，查询商品的名称和分类名称信息：

//数据表模型类联接查询
$itemModel = new \app\model\database\Item();
$result = $itemModel->leftJoin('\app\model\database\Category', 'b', 'a.category_id=b.id')
      				->fields('a.item_id,a.name,a.category_id')
	                ->fields('b.name as category_name')
      				->getAll();

//关联配置联接查询（使用别名）
$itemModel = new \app\model\database\Item();
$result = $itemModel->leftJoin('itemCatery', 'b') //在application/config/model中的item.php配置
      				->fields('a.item_id,a.name,a.category_id')
	                ->fields('b.name as category_name')
      				->getAll();

//关联配置联接查询（不使用别名）
$itemModel = new \app\model\database\Item();
$result = $itemModel->leftJoin('itemCatery') //在application/config/model中的item.php配置
      				->fields('a.item_id,a.name,a.category_id')
	                ->fields('itemCatery.name as category_name')
      				->getAll();



  2、查询条件
      where(where[, alias])
      orWhere(where[, alias])
      其中，where()是AND条件，orWhere是OR条件。
      参数说明：
名称 描述
where 必需。查询条件，参见数据查询中的查询多行记录的condition参数。
alias 可选。表的别名。设置后，会给没有指定别名的字段加上该别名。 

      （1）键值数组条件（系统自动进行安全过滤）

$where = array(
   'user_name' => Request::getGet('username')
);

//查询结果
$itemModel = new \app\model\database\Item();
$userInfo = $itemModel->where($where)
					  ->getRow();


      （2）字符串条件（对变量进行SQL安全过滤）

$userName = Request::getGet('username');

//查询
$itemModel = new \app\model\database\Item();
$userInfo = $itemModel->where("user_name='{$userName}'")
					  ->getRow();


      （3）OR条件（对变量进行SQL安全过滤）

$userName = Filter::sql(Request::getGet('username'));

//查询
$itemModel = new \app\model\database\Item();
$userInfo = $itemModel->where(1)
	->orWhere("user_name='{$userName}'")
	->getRow();



  3、复杂查询条件
      复杂条件还是用where()，orWhere()，having()，orHaving()，只不过要传三个参数。
      函数说明：如果条件数组的比较符号不是等于号“=”时，可使用该方法生成复杂查询条件，并且可以多次使用。
      参数说明：
名称 描述
operator 必需。AND或OR运算符和比较运算符，两者以英文输入法下的“/”分隔，如“OR/in”，"AND/>="。如果不指定OR或AND，则默认为AND，如"in"相当于"AND/in"
field 必需。字段名。
value 必需。字段值。
alias 可选。表的别名。 

      （1）查出ID大于或等于53的商品

$itemModel = new \app\model\database\Item();
$itemData = $itemModel->where('id', '>=', 53)
					  ->getAll();


      （2）查出数组ID值为1,5,33的商品

$itemModel = new \app\model\database\Item();
$itemData = $itemModel->where('id', 'in', array(1,5,33))
					  ->getAll();


      （2）查出出ID大于或等于53，或者数组ID值为1,5,33的商品

$itemModel = new \app\model\database\Item();
$itemData = $itemModel->where('id', '>=', 53)
		->orWhere('id', 'in', array(1,5,33))
		->getAll();



  4、Between条件
      between(field, value1, value2[, alias])
      orBetween(field, value1, value2[, alias])
      其中，between()是AND条件，orBetween是OR条件。
      参数说明：
名称 描述
field 必需。字段名或别名。
value1 必需。第一个值。
value2 必需。第二个值。
alias 可选。表的别名。 

      （1）键值数组条件（系统自动进行安全过滤）

//查询结果
$itemModel = new \app\model\database\Item();
$userInfo = $itemModel->between('id', 1, 10)
					  ->getRow();


      （2）OR条件（对变量进行SQL安全过滤）

//查询
$itemModel = new \app\model\database\Item();
$userInfo = $itemModel->where(1)
    ->where(array('id' => 29))
	->orBetween('id', 1, 10)
	->getRow();



  4、更多条件
      moreWhere(where[, sign])
      函数说明：此方法是在已有的整个where语句最后加上以sign参数指定的符号联接的查询条件，可以多次使用。
      参数说明：
名称 描述
where 必需。查询条件字符串。
sign 可选。查询条件联接符号，默认为“AND”。 

             查出ID为1的商品，如果商品的状态为0则不查询，可以使用status <> '0'，也可以使用IF(status = '0', 0, 1)条件，以下是第二种条件方案：

$itemModel = new \app\model\database\Item();
$itemModel->where(array('id'=>1))
		  ->moreWhere("IF(status = '0', 0, 1)", 'AND')
	 	  ->getRow();


       会在整个Where语句最后加上“ AND IF(status = '0', 0, 1)”的条件。

  5、字段及选项列表
      fields(fields[, alias])
      函数说明：生成字段列表语句，不会进行引号转义，可以多次使用。如果不指定字段列表，默认为“*”。
      如果要添加字段选项，比如Distinct，就用花括号括起来，如：fields('{distinct}')
      参数说明：
名称 描述 
fields
 必需。字段列表字符串或数组。可以是包含一些字段选项，如{distinct}表示去除重复。
alias 可选。表的别名。 

            查出用户user1的用户和邮箱信息。

$userModel = new \app\model\database\User();
$userData = $userModel->where(array('user_name' => 'user1'))
					  ->leftJoin('user_group', 'b', 'a.gid=b.id')
	 				  ->fields('{distinct}') // 字段选项
	  				  ->fields(array('a.id', 'a.name')) //使用别名
	  				  ->fields('id', 'b') //使用表名，框架会自动加上表的别名
      			  ->fields('user_name, email') //不使用表别名
					  ->getRow();



  6、排序
      orderBy(orderBy)
      函数说明：生成排序语句，只能使用一次。
      参数说明：
名称 描述
order 必需。排序语句字符串。 

            查出分类ID的商品，按照添加时间降序排序：

$itemModel = new \app\model\database\Item();
$itemData = $itemModel->where(array('category_id' => $category_id))
					  ->orderBy('create_time DESC')
					  ->getAll();


  7、分组
      groupBy(group)
      函数说明：生成分组语句，只能使用一次。
      参数说明：
名称 描述
group 必需。分组语句字符串，'group by'语句不需要写。 

            统计商品分类的商品数：

$itemModel = new \app\model\database\Item();
$staticData = $itemModel->fields('count(id)	as num')
						->groupBy('category_id')
						->getAll();

  8、分页
      page(limitInfo)   
      函数说明：生成分页语句，只能使用一次。


	

		
注意：
使用page()方法后，返回的结果不再只是数据记录，而是记录总数和数据记录。		
	




      参数说明：
名称 描述
limitInfo 必需。Ocara框架分页插件所生成的分页信息。 


class ListAction extends ItemController
{
     public function __action()
	 {
	    //设置分页
		$this->pager()->config('page', 10, 10);
		
		$itemModel = new \app\model\database\Item();
		$where = array('category_id' => 2);
		
		$result = $itemModel
					->where($where)
					->page($this->pager()->getInfo())
					->getAll();
		
		$total = $result['total'];
		$data = $result['data'];
		$this->pager()->setHtml($total);
	}
}



  9、限制数据行数
      limit(limit[, offset])  
      函数说明：用于限制查询的行数。
      它与page()是互相覆盖的，还有不同处在于，page()返回的结果中，包含total和data两个元素，而limit()直接返回data的数组列表。
      参数说明：
名称 描述
limit 必需。限制数量或开始行。对于Mysql来说，这个参数是LIMIT语句的第一个参数。
offset 可选。限制数量。对于Mysql来说，这个参数是LIMIT语句的第二个参数。Mysql中如果只传递第一个参数，开始行就是0。 


class ListAction extends ItemController
{
     public function __action()
	 {
	    $itemModel = $this->model();
	    $result = $itemModel->fields('item_id, item_name, status');
	     		->where(array('status' => 1))
				->limit(10) //只传第一个参数
				->limit(0, 10) //传两个参数，效果一样
				->getAll();

		print_r($result);
	}
}



  10、规定使用主库查询
      master()   
      如果设置了读写分离数据库配置，使用该方法则规定使用主库进行查询。


$userModel = new \app\model\database\User();
$userModel->where(array('type' => 2))
	->master() //规定使用主库查询
	->getAll();



  12、获取最后一次插入的ID
      getInsertId()  
      函数说明：获取最后一次插入的ID。


class CreateAction extends ItemController
{
     public function __action()
	 {
	    $itemModel = $this->model();
	    $itemModel->data();
	    $itemModel->save();
	    echo $itemModel->getInsertId(); //输出最后插入的item_id值
	}
}



  13、切换数据库
      selectDatabase(name)  
      函数说明：选择数据库，可用于切换Model的数据库。
      参数说明：


	

		名称
		描述
	

	

		name
		必需。新的数据库名。
	





class CreateAction extends ItemController
{
     public function __action()
	 {
	    $itemModel = $this->model();
	    $itemModel->selectDatabase('bbs');
	}
}



  14、切换数据表
      selectTable(name[, primary])  
      函数说明：选择数据表，可用于切换Model的数据表。
      参数说明：


	

		名称
		描述
	

	

		name
		必需。新的数据表名。
	

	

		primary
		可选。主键。复合主键用英文逗号分隔，如"detail,line"。
	





class CreateAction extends ItemController
{
     public function __action()
	 {
	    $itemModel = $this->model();
	    $itemModel->selectTable('item_201702');
	}
}
