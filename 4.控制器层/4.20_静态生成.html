4.20 静态生成


一、什么是静态生成
       在网站应用中，有的页面内容不是经常改变，我们为了使访问速度更快，往往可以将它生成一个静态的html文件，不需要每次都要去动态运行再
输出html，这样也能减少服务器负载。所以，将不是经常修改的动态文件执行结果（如asp，php或jsp文件）生成静态的html文件，用户访问时只要快
速地访问这个静态页面，来提高访问速度，就是静态生成。

二、静态生成的步骤
       Ocara静态生成很简单。
       只需三步:
             （1）配置静态生成的URL路由格式；
             （2）指定要生成的路由和参数数据；
             （3）生成静态页面。

1、确定网站域名配置

       首先要确定和修改application/config/application.php中的网站域名配置WEB_HOST是否正确。
       这个配置就是静态生成和ocUrl()生成URL时使用的域名，如果为空，默认取的是当前访问域名。


    /*
     * 当前网站的主机域名，比如www.baidu.com或bbs.baidu.com等。
     * 建议填写，否则系统会自动获取当前网址中的域名。
     */
    'WEB_HOST' => 'ocara.cn',



2、配置URL路由格式
       静态生成，不是全站都要静态生成，所以，我们在做静态生成时，要确定哪些页面要静态生成。
       我们需要在application/conf/static.php中STATIC配置。

/*
 * ----------------------------------------------------------------------------
 * 开发级配置-静态生成
 * ----------------------------------------------------------------------------
 */
return array(
    'STATIC' => array(
        /**
         * 是否开启静态生成
         */
        'open' => 0,

        /**
         * 静态文件扩展名类型
         */
        'file_type' => 'html',

        /**
         * 分隔符
         */
        'delimiter' => '-',

        /**
         * 路由格式
         */
        'route' => '{c}/{a}-{p}',

        /**
         * 参数格式
         */
        'params' => array(),
    )
);


       其中，open是静态生成开关，file_type是静态生成的文件扩展名，delimiter是参数分隔符。

      （1）配置路由格式（route）
         说明：
              {c}代表控制器，{a}代表动作，{p}表示其他参数。
              路由分隔符只能是下划线"-"和顺斜杠"/"。
         具体实例：
                实例1，假设路由格式为http://域名/控制器/动作/其他参数，配置如下：

'route' => '{c}/{a}/{p}';
                实例2,假设路由格式为http://域名/控制器/动作-其他参数，配置如下：

'route' => '{c}/{a}-{p}';

注意：

模块是不需要配置的，因为模块是强制使用顺斜杠"/"将其与后面的参数分隔的。		 

      （2）配置数据参数格式（params）
         该配置项是多维数组，用于配置URL查询字符串参数的格式。
         说明：
             （a）模块、控制器和动作参数都作为数组键名。
             （b）其他的参数字段都要用花括号括起来。并且以斜杠"/"或英文横杠“-”等分隔。
                       （a1）“/”为目录名分隔符
                                 也就是说其左边参数字符串会作为要保存的目录名称。 比如，若配置成"{id}/{fid}-{tid}"，那么生成静态时会将静态文件保存到id字段命名的目录中。
                       （a2）”-“为文件名参数分隔符（可修改）
                                 可以修改delimiter配置设置文件名分隔符，默认是“-”，必须是英文半角字符。 如下所示：

    /**
     * 文件名分隔符
     */
    'delimiter' => '-',


         具体实例，假设上面的路由格式配置为'{c}/{a}-{p}'，有如下参数配置：
		 

    'params' => array(
        //前台论坛帖子控制器
        'thread' => array(
            'index'   => '{id}-{fid}-{tid}', //参数格式会是：id-fid-tid，访问地址会是http://域名/home/index-1-2-25.html
            'read'    => '{id}-{fid}'				//参数格式会是：id-fid，访问地址会是http://域名//home/read-1-2.html
        ),
        //论坛会员中心模块ucenter
        'ucenter' => array(
            //我的回帖
            'repost' => array(
                'list'=> '{fid}/{tid}'  //参数格式会是：tid，静态页面访问地址会是http://域名/bbs/thread/read/12/2.html
            ),
            //我的帖子
            'thread' => array(
                'list' => '{fid}' //参数格式会是：fid-tid，，静态页面访问地址会是http://域名/bbs/forum/list-2.html
            )
        )
    )
);


3、指定要生成的参数数据
      上面配置了静态生成的路由和参数格式，然后就要用回调函数去指定具体的参数值。
      比如上例的thread控制器。
     （1）函数回调


/**
 * 回调函数
 */
function getHtmlData($module, $controller, $action, $params)
{
     if ($controller == 'thread') {
           if ($action == 'list')  {
          		 $model = new app\model\database\ThreadModel();
                 //字段映射
                 foreach ($params as $key => $field) {
                       if ($field == 'fid') {
                             $params[$key] = 'forum_id AS fid';
                       }
                 }
                 return $model->fields($params)->getAll();
            } elseif ($action == 'read') {
                 //read动作的代码
            }
     } else {
           //其他代码
     }

 	 return array();
}


      （2）类方法回调

 * 回调方法
 */
class StaticCallBack

     {

     }

4、生成静态页面

      通过以上两步，配置并写好了回调函数，下面我们只要调用生成函数，就能生成静态了。

      （1）目录权限
            请保证生成的目录public/static有可写权限，生成的静态页面都是在该目录下面。
	  
      （2）生成静态
            生成函数，就是使用系统插件StaticBuilder的genAll()方法，格式如下：
            genAll(callback)
            参数说明：
          


    

        名称
        描述
    

        

        callback
        必需。回调函数、类的方法。
    




            callback传递的参数值有三种：

            （a）回调函数


class staticGenerateController extends Controller
{
    public function actionGenerate()
	{
        $this->staticBuilder->genAll('getHtmlData');
    }
}



            （b）回调对象方法


class staticGenerateController extends Controller
{
    public function actionGenerate()
	{
        $handler = new StaticCallback();
        $this->staticBuilder->genAll(array($handler, 'getHtmlData'));
    }
}



            （c）回调对象方法

                  如果getHtmlData是静态方法，则使用StaticCallback::getHtmlData。


class staticGenerateController extends Controller
{
    public function actionGenerate()
	{
        $this->staticBuilder->genAll('StaticCallback::getHtmlData');
    }
}



      5、访问静态页面

            使用ocUrl()生成链接地址时，系统会根据静态配置规则查找静态链接的文件是否存在，存在的话直接访问，否则会访问动态地址。


    $url = ocUrl('thread/list', array(
        'fid' => 12,
        'tid' => 123,
    ));

    //如果生成了静态链接：http://ocara.cn/thread/list/12-123.html，则会直接访问，否则会访问动态地址：http://ocara.cn/thread/list/fid/12/tid/123
    header('location:' . $url);
