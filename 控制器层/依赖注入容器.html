依赖注入容器


      依赖注入容器，可以实现类或接口的绑定、实例化或实现，以及依赖管理。

1、容器的使用
      （1）默认容器
            容器类为\Ocara\Core\Container。
            框架开始时便生成了一个默认容器，可以通过以下两种方式获取:
            （a）Container::getDefault()。
                  实例如下：

//Container::getDefault()
$container = Container::getDefault();


            （b）使用ocContainer()函数。
                  实例如下：

//继承自Base的类中使用
class test extends \Ocara\Base
{
    public function output()
    {
       $response = ocContainer()->response;
    }
}



     （2）私有容器
            控制器动作继承于ServiceProvider类，所以其本身包含自己的私有容器$this->_container。
            可以使用“$this->属性名”方式获取默认容器中的类对象。
            如果默认容器和私有容器有同名的类对象，这种调用方式会优先调用私有容器中的对象，如果没有则取默认容器中的。
            实例如下：

class EditAction extends ItemController
{
    /**
     * 注册对象
     */
    public function register()
    {
       $this->_container->bindSingleTon('response', '\Base\MyResponse'); //私有容器中注册对象
    }

	/**
	 * 初始化
	 */
	public function __action()
	{}

	public function registerForms()
	{}

	public function display()
	{
        $id = $this->request->getGet('id'); //使用默认容器的request对象
    }

	public function submit()
	{
        $data = $this->response->setStatusCode(400); //优先使用当前控制器私有容器$this->_container的response对象
    }
}


      （2）生成新的容器
            除了使用默认容器，我们还可以新建一个容器，只要新建\Ocara\Core\Container类即可。
            如下所示：

//使用new 关键字新建容器
$container = new \Ocara\Core\Container();

//使用build()方法新建容器
$container = \Ocara\Core\Container::build();



2、类或接口的绑定
      （1）动态绑定
            动态绑定使用容器的bind()方法，也可以使用容器的属性来操作。
                  bind(name, source[, params, deps])
            参数说明：
名称 描述
name 必需。绑定名称或类名、接口名。
source 必需。实例化来源。可以是类和方法的数组、回调函数或类名字符串。
params 可选。类的构造函数的前面部分参数。会与get()方法中传递的后面函数合并。
deps 可选。属性注入数组。键名是属性名称，键值是注入的对象。
使用前提是类中必须包含该属性名称的set和get方法。
比如，Computer类中要使用属性注入的方式向它的mouse属性为一个Mouse类，需要在它内部定义一个setMouse()和getMouse()方法。 
            实例如下：
            （a）绑定和使用

//方法数组
Base::$container->bind('userChecker', array('User', 'getChecker'));

//回调匿名函数
Base::$container->bind('userChecker', function(){
    return User::getChecker();
});

//获取类名
Base::$container->bind('userChecker', '\User\Checker');

//用类实现接口
Base::$container->bind('\Interface\User\Checker', '\User\Checker');

//用新类替换被绑定的类（新类须继承旧类，或者与旧类继承同一个父类）
Base::$container->bind('\User\Checker', '\User\NewChecker');

            另外，可以不用bind()方法，直接使用未定义的属性来绑定。实例如下：

//方法数组
Base::$container->userChecker = array('User', 'getChecker');

//回调函数
Base::$container->userChecker = function(){
    return User::getChecker();
};

//获取类名
Base::$container->userChecker = '\User\Checker';

            （b）传递参数
                    传递参数可以是在bind()绑定时，或者get()获取时，两者会合并使用。

class Computer
{
    protected $_mouse;

    public function __construction(\Mouse $mouse, $name, $config = array())
    {
        $this->_mouse = $mouse;
    }
}

                    使用时代码如下实例：

//绑定时传递前面的参数（注入鼠标对象）
$container = new \Ocara\Core\Container();
$mouse = new \Mouse();
$container->bind('computer', '\Computer', array($mouse));

//获取对象时传递后面的参数（注入键盘对象）
$keyboardObj = new \Keyboard();
$name = 'computer1';
$config = array();
$computer->get('computer', array($name, $config));

                    上述实例中，$mouse参数是通过传参传过去的，但是也可以不传而让容器自己自动重新建一个\Mouse类，这称为依赖管理。
                    但是前提是容器中也指定了该类的实现，而参数数组中该参数使用null即可。如下所示：

//绑定鼠标对象
$container->bind('\Interface\Mouse', '\Mouse');

//绑定时传递前面的参数（注入鼠标对象）
$container = new \Ocara\Core\Container();
$container->bind('computer', '\Computer', array(null)); //这里传null

//获取对象
$keyboardObj = new \Keyboard();
$name = 'computer1';
$config = array();
$computer->get('computer', array($name, $config));

                    上面定义了\Interface\Mouse接口的实现类为\Mouse，在传参时只要传递null即可，容器会自动创建该\Mouse类传递给Computer的构造函数。
                    实现自动化依赖管理。


            （c）属性注入
                     对于第4个参数deps，它是一个属性注入数组，被注入的类必须包含set和get方法，如下所示：

class Computer
{
    protected $_mouse;
    protected $_container;

    public function __construction($_container)
    {
        $this->_container = $_container;
    }

    /**
     * 鼠标
     */
    public function setMouse($mouseObj)
    {
        $this->_mouse = $mouseObj;
    }

    public function getMouse()
    {
        return $this->_mouse;
    }

    /**
     * 键盘
     */
    public function setMouse($mouseObj)
    {
        $this->_mouse = $mouseObj;
    }

    public function getMouse()
    {
        return $this->_mouse;
    }
}

                     注入代码如下实例：

//绑定时属性注入（注入鼠标对象）
$mouseObj = new \Mouse();
$container->bind('computer', '\Computer', array($container), array('mouse' => $mouseObj));

//获取对象时属性注入（注入键盘对象）
$keyboardObj = new \Keyboard();
$computer->get('computer', array(), array('keyboard' => $keyboardObj));


      （2）单例模式绑定
            与动态绑定区别在于：动态绑定的类可以通过容器的create()方法多次新建一个实例，但是单例模式绑定的则不能，只会在第一次使用时自动create()一次。
                  bindSingleton(name, source[, params, deps])
            参数和实例同上。


3、绑定类的使用
      使用容器的get()方法，或者使用容器绑定过的名称属性，就可以获取实例化后的绑定类。
      （1）使用get()方法
                  get(name[, params, deps])
            参数说明：
名称 描述
name 必需。绑定名称或类名。
params 可选。类的构造函数的后面部分参数。会与bind()或bindSingleton()中指定的前面参数合并
deps 可选。属性注入数组。同上bind()或bindSingleton()中的deps参数，最后会将两者进行合并使用。 
            实例如下：

$userChecker = ocContainer()->get('userChecker');

      （2）使用未定义的属性
            使用容器中未定义的绑定名称属性， 也可以实现在同样的效果。
            上述实例等同于：

$userChecker = ocContainer()->userChecker;
