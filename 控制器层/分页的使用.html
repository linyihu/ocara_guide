分页的使用


1、分页配置
      框架默认的分页类是Pager，如果要使用自己的分页插件，可以将分页配置选项class_name配置成该分页类的名称。
      目前，分页配置有4个选项，如下所示：

/*
 * 分页配置
 */
$CONF['PAGE'] = array(
	'class_name' 	=> '', 		  //分页类名称，默认是Pager
	'page_param' 	=> 'page',	  //URL查询字符串中分页参数名称
	'per_page'	 	=> 10,		  //每页显示多少条记录
	'per_show'   	=> 10		  //一次显示多少页
);


2、默认分页插件的使用
      如果没有配置分页类名，则框架会使用Pager类。该分页插件的方法介绍，可以参见扩展插件/分页。
      这里介绍一下它的使用步骤：
      （1）获取分页类对象
              一般使用控制器的pager()方法来获取当前分页对象，即上面所述配置class_name指定的分页类，如果没有配置，则默认是Pager。
      （2）设置分页参数
               可以使用分页类的config()方法来设置分页参数名称、每页显示多少条记录和一次显示多少个页码。也可以使用相应的独立方法，分别
               是setPageParam()、setPage()、setPerge()三个方法。
      （3）进行数据库分页
              分页一般用于数据库记录查询。数据库查询分页，主要是通过数据模型Model的page()方法分设置分页信息，类似于Mysql的limit语句
              的值，即起始索引和偏移值：limit $start, $number。
      （4）生成分页HTML并显示
              分页HTML生成，一般使用分页类的setHtml()方法。显示时，只要将当前分页对象用模板变量使用。
              Pager的属性基本上都是公有属性（public）， 可以自由使用，包括上一页、下一页等等页码。
	  		  
      具体实例如下：

class ListAction extends ItemController
{
     public function _action()
	 {
	    //获取分页对象  
		$pager = $this->pager();
        
        //设置分页参数
		$pager->config('page', 10, 10);
		
		//进行数据库分页查询
		$itemModel = $this->model('\Model\Item');
		$where = array('category_id' => 2);
		
		$result = $itemModel->where($where)
			->page($pager->info())
			->getAll();
		
		$total = $result['total'];
		$data = $result['data'];
		
		//生成分页HTML
		$pager->setHtml($total);
		
        //作为模板变量使用
		$this->assign(
			'data' => $data,
			'pager' => $pager
		);
	}
}

3、自定义分页插件
      如果要使用自己的分页插件，上述也讲到了，只要配置好class_name选项即可。
      分页类中可以使用上述配置，也可以自己新建配置选项。
      但是请注意，数据库查询时要通过Model的page()方法传递分页信息，并且使用该方法后，数据查询结果返回的不再是记录结果数
      组,而是多返回了个记录总数，即包含了total和data两个数组元素，其中data才是记录结果数组。
