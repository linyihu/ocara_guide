角色权限控制

       一般一个应用中都会涉到及角色的权限控制。例如网站后台一般都要给不同的管理员组或管理员分配不同的权限。 
       本框架做权限控制，很简单，只要开发者使用Auth设置好权限，然后提供一个回调函数调用check()方法检测权限即可。

一、权限分配

1、编辑角色
    角色信息一般保存在数据库中。大多情况下，角色一般是会员组，管理员组等。这里的角色组是为了设置权限用，需要先从数据库中查出
来，这里就不用说明了。

（1）设置角色
      角色的编辑使用Auth的setRole()方法，格式如下：
            setRole(roleList)
      参数说明：
名称 描述
roleList 必需。角色层级列表。也就是要设置的角色，多层级角色用英文半角状态下的圆点“.”分隔，或者直接用数字型数组。 

      实例如下：

$auth->setRole('role1');

      如果增加子角色，只要用圆点"."或者用数组，即：

//使有圆点键名方式
$auth->setRole('role1.role2'); 

//使用数组方式
$auth->setRole(array('role1', 'role2'));

      这相当于设置$auth的私有属性（private）$_data属性值，如下代码：

$auth->_data['role1']['role2'] = array();

（2）删除角色
      删除角色只要使用delRole()方法，格式如下：
            delRole(roleList)
      参数同上，使用方法同上。
	  
2、分配权限
（1）设置权限
      权限的设置使用Auth的setAuth()方法，格式如下：
            setAuth(roleList, authList)
      参数说明：
名称 描述
roleList 必需。同上。
authList 必需。要设置的权取。一般是权限路由列表，或者数据表的权限ID。如果是权限路由列表，即模块/控制器/动作路由，多层级之间也用英文半角状态下的圆点“.”分隔，或者直接用数字型数组。 

      实例如下：
      权限分配，可能有下面两种情况。
	  
      （a）直接定义权限。如下设置role1的role2拥有bbs模块的forum控制器的index权限：

//使用圆点键名方式
$auth->setAuth('role1.role2', 'bbs.forum.index'); 

//使用数组方式
$auth->setAuth('role1.role2', array('bbs', 'forum', 'index'); 

             结果相当于设置$auth的$_data属性值，如下所示：

$auth->_data['role1']['role2']['bbs']['forum']['index'] = null;


     （b）权限对象可能是数据库中的权限ID字段。如下示例：

$auth->setAuth('role1.role2', '1');

             结果相当于如下代码：

$auth->_data['role1']['role2'][1] = null;

 
（2）删除权限
      删除权限只要使用delAuth()方法，格式如下：
            delAuth(roleList, authList)
      参数同上，使用方法同上。
	  
3、获取权限
      获取权限使用Auth的getAuth()方法，格式如下：
            getAuth([roleList])
      实例如下：

//获取所有权限
$allAuthData = $auth->getAuth();

//获取role1.role2的权限设置
$authData = $auth->getAuth('role1.role2');


	  
4、初始化权限数据
      不仅可以使用上述方法设置权限，我们也可以使用setData()方法直接设置好自定义的权限数组，格式如下：
            setData([data])
      参数说明：
名称 描述
data 必需。权限数组。一般都是通过本插件Auth的getAuth()方法缓存的数组修改后再设置回去，或者如果了解数组格式的话，就自定义该数组。 
      实例如下：

//自定义权限数组
$model = new Auth();
$data = $model->getAuthData();
//设置权限
$auth = new Auth();
$auth->setData($data);




5、检测权限
      要检测权限，使用Auth的check()方法，格式如下：
            check(roleList, authList))
      参数说明：
名称 描述
roleList 必需。同上setAuth()。
authList 必需。同上setAuth()。 
       
      实例如下：
	   
      函数实例
            检测role1角色下的role2角色是否拥有bbs模块的forum控制器下面的index动作的权限，如下：
            （a）直接指定权限分配的对象，

//使用圆点键名方式
$result = $auth->check('role1.role2','bbs.forum.index');

//使用数组方式
$result = $auth->check('role1.role2', array('bbs', 'forum','index'));
            （b）如果权限分配的对象是以数据库形式保存的，比如bbs模块forum控制器index动作对应的记录id为1，检测权限则为：

$result = $auth->check('role1.role2', 1);


二、检测权限步骤

1、设置检测回调函数
      在应用根目录下的resource/conf/develop.php中存在"回调函数配置"的注释和配置，可以配置其中的权限控制回调路径：

$CONF['CALLBACK']['auth'] = array(
	'check' => 'auth/check', //权限控制检测回调函数或路由
	'invalid_error' => '', //无权限的界面提示
);

       一般的，在调用配置的回调函数或路由的时候，框架会自动传递当前路由参数，也可以使用函数或类的方法。
2、编写检测函数
      上面配置好了检测路由，就可以编写该检测路由了，即auth控制器的check动作，如下所示：

class CheckAction extends AuthController
{
	/**
	 * 初始化
	 */
	protected function _action($module, $controller, $action)
	{
		//获取当前的角色-自定义方法
		$role = $this->getRole();
		
		//获取权限数据-自定义方法
		$model = new \Model\Auth();
		$authData = $model->getAuthData();
		
		$auth = new Auth();
        $auth->setData($authData);
		
		//检测权限
		$result = $auth->check($role, array($module, $controller, $action));
		
		return $result;
	}
    
    protected function getRole()
    {
        //这里获取角色
    }
    
    protected function getAuthData()
    {
        //这里查询数据库或文件，返回权限数据
    }
}

       说明：
                 （1）一般地实际在做权限控制时，角色和权限数组都保存在数据库中，所以如果要检测所有角色的权限，可以查询数据库。
                       getRole()是自定义的获取角色的方法。
                 （2）编写完检测函数后，Ocara会调用auth控制器的check动作检测当前角色用户对当前控制器动作的权限，如果没有权限，则会抛出异常提示。
