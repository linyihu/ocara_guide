资源提供器


一、资源提供器介绍
          Ocara不仅提供了事件机制进行业务回调处理，用于解耦。
          但是有时候我们可能还需要通过回调函数返回一些数据。所以资源提供器派上用场了。
        （1）资源提供器与服务提供器的区别
                服务提供器是一种服务组件类对处提供服务，资源提供器是一种一般用于事件回调的中间件处理器，重在返回数据。
        （2）资源提供器与事件的关系
                资源提供器一般是通过事件触发的。
        （3）资源提供器与中间件的关系
                资源提供器的父类Ocara\Core\Resource继承于中间件类，所以它也是一种中间件。
二、使用方法
        首先要新建一个资源提供器类，它需继承Ocara\Core\Resource，必须提供handler()处理函数。
        上面已说明资源提供器也是中间件，所以必须实现handler()函数，并在事件触发时会自动调用。

namespace Resource;

use Ocara\Core\Resource;

class CacheGetConfig extends Resource
{
    public function handler($params)
    {
        if ($params['type'] == 'redis') {
            return array(
                'type' => 'redis',
                'host' => '127.0.0.1',
                'port' => 6379
            );
        } else {
            /*...*/
        }
    }
}



    然后在使用的地方调用：

use Ocara\Core\Base;

class CacheFactory extends Base
{
    //事件注册
    public function registerEvents()
    {
        $this->event('get_config')
             ->append('\services\resources\CacheGetConfig', array('type' => 'redis')));
    }


    /*...*/

    //事件触发
    public function getConfig()
    {
        $cacheConfig = $this->fire('get_config'); //会自动调用handler方法
    }
}
