1.8 资源提供器


一、资源提供器介绍
       在有些地方，我们需要通过事件的回调函数获取一些数据，这种称为资源提供器。

二、使用方法
        事件的回调处理函数可以添加多个，但是作为资源提供器时，只需要一个回调函数。
        这时只要使用事件的resource()方法声明为资源事件即可。
              resource([isResource])



	

		名称
		描述
	

		

		isResource
		可选。指定是否为资源事件。
	






class OneCache extends Resource
{
    const EVENT_GET_CONFIG = 'getConfig';

    /**
     * @throws Exception
     */
    public function registerEvents()
    {
        $this->event(self::EVENT_GET_CONFIG)
            ->resource() //声明事件为资源事件
            ->append(ocConfig('RESOURCE.one_cache.get_config', null));
    }

    public function connect(serverName)
    {
        //资源事件触发
        $config = $this->fire(self::EVENT_GET_CONFIG, array($serverName)); 
        
        // 这里写其他代码 ... ...
    }
}




三、资源提供器配置
        类似于事件配置。资源提供器可以在resource.php中配置。目前框架默认提供了几个配置。
        比如缓存配置获取资源提供器的配置：


        /*
         * 缓存相关
         */
        'cache' => array(
            'get_config' => '', //获取缓存配置事件回调
        ),
